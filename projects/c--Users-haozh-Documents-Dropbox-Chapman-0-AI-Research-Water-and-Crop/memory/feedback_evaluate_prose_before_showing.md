The user repeatedly caught ambiguous or dangling phrasing I presented without checking it first ("in the first half of what?", "this per-year number is unclear", "you don't follow my revision order"). The recurring failure is showing unevaluated first-draft prose.

**Why:** handing over wording that has obvious referent/clarity bugs wastes the user's time, makes them do my QA, and erodes trust. The user explicitly asked: "can you double check what you write, evaluate its quality before showing to me?"

**How to apply:** after writing ANY sentence-level edit, before showing it, run a quick quality pass —
1. Every "this / that / it / the first half / the period / those" has an unambiguous referent in the same or adjacent clause. Deleting a noun phrase often orphans a later reference (e.g. cutting "of the sample period" left "the first half" dangling).
2. No phrase dangles after a deletion or a merge.
3. Numbers, windows, and percentages are internally consistent (if the averaging window changes, the implied % change must change too).
4. It reads cleanly aloud, and it actually matches the structure/length the user dictated (one paragraph means one paragraph, not two).

Only then present, and when the user has flagged a quality problem, show the self-evaluation so they can see it was checked. Applies across all research projects. Related: [[feedback_topic_sentence_first]], [[feedback_clean_keeps_green]].
