The auxiliary-parameter sensitivity analysis (manuscript sec:sensitivity line 674 + Appendix Figures sens_beta/eta/hydro) was rebuilt for the 8-cat construct on 2026-06-27, replacing the 9-cat sandbox `T1p3_sens_*` sprawl.

**Production programs (in the pipeline as senssweep / sensfigs):**
- `program/2_structural/sensitivity_cf.py` — 6-point sweep: β∈{0.85,0.95}, η∈{0.15,0.30}, aquifer kernel (β_0,β_1) ±1 SE. Reuses `bootstrap_cf.run_cf_for_draw` (the tested grid-runner). Central case is READ from `cf_results.json`, never recomputed. Modes: `--launch` / `--point <name>` / `--combine`. Writes ONLY to `results/sensitivity/`.
- `program/3_exhibits/build_sensitivity_figs.py` — reads `results/sensitivity/sensitivity_results.json` → `latex/figures/sens_beta.pdf`/`sens_eta.pdf`/`sens_hydro.pdf` + `latex/tables/sensitivity_macros.tex` (the line-674 ranges). Uses the **3×3 grouped-bar layout** (rows = GW regime open/tax/cap, columns = SW institution autarky/bilateral/centralized, low/central/high bars per panel) — ported from the old `T1p3_sensitivity_figs.py` after a first single-panel version was rejected as cluttered. NOT auto-detected by build_pipeline (its input lives under results/sensitivity/), so it's documented in PIPELINE_manual only.

**β AND η both require re-estimation (η added 2026-06-28), not just a CF override.** Both enter the structural estimation, so each sweep point loads θ re-estimated at that value (isolated, never the central). β enters only the Euler discounting → re-run `stage2_euler.py`+`stage2_ols.py` with env `BETA_OVERRIDE` → `structural_params_beta85/95.json`. η enters the water cost AC, which feeds the WHOLE chain, so its re-estimation re-runs the full FIVE programs `calibrate_water_cost`→`build_estimation_sample`→`stage1_ccp`→`stage2_euler`→`stage2_ols` under env `ETA_OVERRIDE` (rescale cost_gw by η/0.234, in-memory, never touches `district_gw_head.dta`) + `EST_SUFFIX` (suffix the AC/sample/CCP chain + stage2's INPUT reads) + `EST_OUT_SUFFIX` (stage2 OUTPUT: moments/params/vcov/tables) → `structural_params_eta15/30.json`; the η CF point then loads that θ AND rescales the engine `cgw_coef`. `ensure_eta_params(eta)` in sensitivity_cf.py runs the chain. Suffix logic: EST_SUFFIX=input chain, EST_OUT_SUFFIX=stage2 output (falls back to the BETA-derived `_beta` label when unset, so the β path and canonical run are byte-identical). Diagnostics (heatmaps, CCP-fit pdf, logs) are gated/suffixed on EST_SUFFIX so isolated runs never clobber them. The aquifer kernel does NOT enter the estimation moment (perceived-stationary design) → hydro stays a pure in-memory override with θ̂ fixed. Isolation proven by MD5 of 9 canonical files (cf_results, structural_params, structural_vcov, euler_moments, first_stage_ccp, AC_kt_baseline, estimation_sample, Table-3-params, bootstrap_ci) before+after the whole run — all BYTE-IDENTICAL. See [[estimation_pipeline_E1_E2_checked_data]] and [[deferred_repoint_8cat_revenue_cost]].

**Aquifer β_1 drifted:** the 8-cat calibration is β_1 = 0.0130/mi (≈53-mi half-life), NOT the old 0.025 (28-mi) that the sandbox hardcoded. `sensitivity_cf.py` reads β_0, se_beta_0, β_1 LIVE from `inputs['elas']` so it can't go stale; β_1 (grid-calibrated, no closed-form SE) is perturbed by β_0's relative SE (~13%) as a ±1-SE-equivalent. Supersedes the 0.025 figure in [[aquifer_elasticity_calibration]].

**Finding (8-cat result):** centralized-tax is the highest-welfare pair at all 7 columns ($2.6–$2.9bn; the $2.6 min is the re-estimated η=0.15 cell, 2648 — was $2.7 under the CF-only η override); within-regulated ordering preserved everywhere; autarky-open positive across all six ($0.52–$1.10bn around $0.80bn) and exceeds centralized-open at ALL six points. The old "five of six / single reversal at β=0.85" claim is GONE (now 6/6) — line 674 updated accordingly. Re-estimating θ for η (vs the earlier CF-only override) shifts the η welfare cells by ~1–2% and β_W by only ±0.7% (water-cost preference robustly identified; β_R varies ~25%, monotonic in η). η-rescale moves GW-district AC (51→45 at η=0.15, 51→57 at η=0.30) and leaves SW-district AC untouched (η is GW-only), confirming the override is real.
