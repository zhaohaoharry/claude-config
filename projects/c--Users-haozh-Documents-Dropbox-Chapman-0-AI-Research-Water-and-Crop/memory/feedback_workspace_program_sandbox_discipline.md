Standing workspace discipline the user wants (stated 2026-06-19, Water and Crop; generalize to all
research projects). **Why:** avoids the "decision made but never implemented" failure and stops
experiments from silently contaminating main results.

**`program/` = production.** Subfolders organized **by program type**. Inside each subfolder, name
programs by **order of first use + economic role**, with **decimal numbering** so a program inserted
between 1 and 2 becomes 1.5 and run order is readable from the numbers (the existing "Stage 1.5"
follows this). Linkages recorded in code-derived `PIPELINE.md` (see
[[feedback_code_derived_pipeline_manifest]]).

**`sandbox/` = experiments only.** Data generated by a sandbox program **stays in sandbox under a
distinct name** (e.g. an in-folder `outputs/`) and **never writes to main `results/` or `latex/`**.
An experiment is promoted only by an explicit decision; then the main-folder program, `PIPELINE.md`,
and the main-folder data all change together.

**How to apply.** When running an experiment, suffix its artifacts distinctly AND point its output
paths into the sandbox experiment folder, not `results/`/`latex/`. When a decision is made, log it
immediately in the project-root `DECISIONS.md` (which `PIPELINE.md` links to) and implement it
(e.g. rewrite the affected prose) in the same session.

**Worked failure (2026-06-19):** the `decision_v2_variant` re-estimation leaked 12 `*_didv2`
artifacts into main `results/` and `latex/`, and a `_v2` output-suffix collided with a pre-existing
B4 welfare-ledger experiment. Fix: relocated all outputs into `program/sandbox/decision_v2_variant/
outputs/`, redirected every variant script so it can never write to main, and renamed the whole
experiment to a unique `_didv2`.

**Migration EXECUTED 2026-06-19 (Water and Crop), cluster-safe.** 41 Python production scripts moved
out of flat `program/sandbox/` into `program/1_data/` (18), `program/2_structural/` (8, the
import-coupled estimation→counterfactual cluster kept together so bare imports resolve),
`program/3_exhibits/` (15); 3 reduced-form `.do` moved to `program/stata/`. Each folder has a README;
`PIPELINE.md` is authoritative order. **Naming caveat (generalizes):** decimal numeric prefixes can't
be retrofitted onto import-coupled or invoked-by-name modules — a numeric prefix makes a module
non-importable. Keep importable names for those (their `A5/A10/step1p5..step5` prefixes already encode
order); apply decimal numbering to NEW standalone scripts only. **Always full-backup the code before a
no-git move** (`quality_reports/backups/program_code_pre_reorg.zip`), then smoke-test: import every
moved module + re-run a cheap generator byte-identical + audit 0 orphans + manuscript compiles. The
provenance tools (`build_pipeline.py`, `audit_provenance.py`) must skip experiment dirs and `*_didv2`
(suffix-collision mis-attributes variants to canonical artifacts). See
[[decision_id_v2_vs_fine_impact]], [[transition_table_estimation_rebuild]],
[[feedback_every_number_has_provenance]].
